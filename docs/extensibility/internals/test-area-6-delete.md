# <a name="test-area-6-delete"></a>테스트 영역 6: 삭제
이 소스 제어 플러그 인 테스트 영역 삭제 동작에 설명 합니다.  
  
 소스 제어에서 작업을 삭제 하 응답 **솔루션 탐색기**합니다.  
  
 다음은 삭제할 수 있는 항목의 목록입니다.  
  
-   파일  
  
-   폴더  
  
-   프로젝트  
  
 프로젝트 형식에 따라 옵션을 할 수 있습니다 **제거** (디스크에 파일을 리프) 프로젝트 또는 **삭제** (디스크에 파일을 제거 합니다.) 프로젝트. 두 동작은 모두 제거 프로젝트 또는 항목에서 **솔루션 탐색기**합니다.  
  
## <a name="expected-behavior"></a>예상 되는 동작  
 Delete 테스트 영역에 테스트 사례에 대 한 예상 되는 동작은입니다.  
  
-   삭제 된 항목 내에 표시 되는 더 이상 **솔루션 탐색기**합니다.  
  
-   삭제 된 프로젝트 또는 항목의 부모를 체크 아웃 필요에 따라 (가능한 프롬프트.)  
  
-   체크 아웃 삭제 또는 추가 된 항목 후에 나타나지 않습니다는 **보류 중인 체크 인** 창.  
  
-   항목이 아직 소스 제어 저장소 내는 삭제 후에 있고 수동으로 제거 해야 합니다.  
  
|작업|테스트 단계|예상된 결과를 확인합니다|  
|------------|----------------|--------------------------------|  
|클라이언트 프로젝트를 삭제 합니다.|1.  클라이언트 프로젝트를 만듭니다.<br />2.  소스 제어에 솔루션을 추가 합니다.<br />3.  전체 프로젝트를 솔루션에서 제거|예상 되는 일반적인 동작입니다.|  
|빈 파일을 삭제 합니다.|1.  클라이언트 프로젝트를 만듭니다.<br />2.  0 바이트 파일을 프로젝트에 추가 합니다.<br />3.  소스 제어에 솔루션을 추가 합니다.<br />4.  파일을 선택, 삭제 합니다.|예상 되는 일반적인 동작입니다.|  
|파일을 하나 포함 한 폴더를 삭제 합니다.|1.  단일 프로젝트 솔루션을 만듭니다.<br />2.  폴더를 추가 합니다.<br />3.  폴더에 파일 하나를 추가 합니다.<br />4.  소스 제어에 솔루션을 추가 합니다.<br />5.  표시 되는 메시지를 방지 하기 위해 프로젝트 확인해 보세요.<br />6.  폴더를 삭제 합니다.|예상 되는 일반적인 동작입니다.|  
|파일 시스템 웹 프로젝트 삭제|1.  파일 시스템 웹 프로젝트 (사용 하 여 UNC 경로 지정 하려면 찾아보기 단추를) 만듭니다.<br />2.  소스 제어에 솔루션을 추가 합니다.<br />3.  솔루션에서 전체 프로젝트를 제거 합니다.<br />4.  로컬 웹 프로젝트에 대해 1 ~ 3 단계를 반복 합니다. (코드를 통해 서로 다른 경로 실행 하지만 같은 외부 인터페이스가 및 동작을 갖는).|예상 되는 일반적인 동작입니다.|  
|파일 시스템 웹 프로젝트에서 파일을 삭제 합니다.|1.  파일 시스템 웹 프로젝트를 만듭니다.<br />2.  소스 제어에 솔루션을 추가 합니다.<br />3.  프로젝트에서 파일을 삭제 합니다.<br />4.  로컬 웹 프로젝트에 대해 1 ~ 3 단계를 반복 합니다. (코드를 통해 서로 다른 경로 실행 하지만 같은 외부 인터페이스가 및 동작을 갖는).|예상 되는 일반적인 동작입니다.|
