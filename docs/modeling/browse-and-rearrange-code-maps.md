# <a name="browse-and-rearrange-code-maps"></a>코드 맵 찾아보기 및 다시 정렬
코드 맵의 항목을 다시 정렬하여 읽기 쉽게 만들고 해당 성능을 향상시킵니다.

 솔루션의 기본 코드에 영향을 주지 않고 코드 맵을 사용자 지정할 수 있습니다. 이 기능은 주요 코드 요소에 중점을 두거나 코드에 대한 아이디어를 전달하려는 경우에 유용합니다. 예를 들어 관심 있는 영역을 강조 표시하기 위해 맵에서 코드 요소를 선택하고 필터링한 다음 코드 요소와 링크의 스타일을 변경하고 코드 요소를 숨기거나 삭제하고 속성, 범주 또는 그룹을 사용하여 코드 요소를 구성할 수 있습니다.

 **요구 사항**

-   코드 맵을 만들려면 Visual Studio Enterprise가 있어야 합니다.

-   Visual Studio Professional에서 코드 맵을 보고 제한적으로 편집할 수 있습니다.

##  <a name="ManageLargeGraphs"></a> 코드 맵 작업을 시작
 코드 맵을 만듭니다 (참조 [솔루션 전체의 종속성 매핑](../modeling/map-dependencies-across-your-solutions.md) 자세한 세부 정보에 대 한). 생성 완료를 클릭 합니다. 지도 대해 기다려야 않으려면는 **취소** 언제 든 지 생성 프로세스를 중지 하는 링크입니다. 그러나 이렇게 하면 모든 종속성 및 링크의 세부 정보가 표시되지 않습니다.

 맵을 생성한 후 다음 팁에 따라 시작 코드를 검토합니다.

-   코드에서 자연 종속성 클러스터를 확인합니다. 맵 도구 모음에서 선택 **레이아웃**, **빠른 클러스터**![그래프 도구 모음의 빠른 클러스터 단추](../modeling/media/quickclustersicon.gif)합니다. 참조 [지도 레이아웃 변경](#Selecting)합니다.

     ![종속성 그래프 &#45; 빠른 클러스터 레이아웃](../modeling/media/dependencygraph_quickclusters.png)

-   관련 노드를 그룹화하여 맵을 더 작은 영역으로 구성합니다. 자동으로 표시되는 그룹 간 종속성만 표시하려면 해당 그룹을 축소합니다. 참조 [노드 그룹](#OrganizeGroups)합니다.

-   필터를 사용하여 맵을 단순화하고 관심 있는 노드 또는 링크의 유형에 집중합니다. 참조 [노드 및 링크 필터링](#FilterNodes)합니다.

-   큰 맵의 성능을 최대화합니다. 참조 [솔루션 전체의 종속성 매핑](../modeling/map-dependencies-across-your-solutions.md) 자세한 정보에 대 한 합니다. 예를 들어 켜기 **빌드 건너뛰기** 지도에 있는 항목을 업데이트 하는 경우 Visual Studio 솔루션을 다시 빌드합니다 하지 않는 하므로 맵 도구 모음에서 합니다.

##  <a name="Inspect"></a> 노드 및 링크를 검사 합니다.
 코드 맵에는 다음 종류의 링크가 표시됩니다.

-   개별 링크는 두 노드 간의 단일 관계를 나타냅니다.

-   그룹 간 링크는 서로 다른 그룹에 속한 두 노드 간의 관계를 나타냅니다.

-   집계 링크는 두 그룹 간에 동일한 방향을 가리키는 모든 관계를 나타냅니다.

> [!TIP]
>  기본적으로 맵은 선택한 노드에 대한 그룹 간 링크만 표시합니다. 이 동작을 표시 하거나 숨기려면 그룹 간 집계 된 링크를 변경 하려면 **레이아웃** 코드에 도구 모음을 매핑한 선택 **고급**, 다음 **그룹 간 링크 모두 표시** 또는 **모든 그룹 간 링크 숨기기**합니다. 참조 [숨기기 또는 표시 노드 및 링크](#HidingShowing) 내용을 확인 합니다.

|그룹의 콘텐츠를 표시하거나 숨깁니다.|-그룹을 확장 하는 노드에 대 한 바로 가기 메뉴를 열고 선택한 **그룹**, **확장**합니다.<br />     또는<br />     펼침 단추(아래쪽 화살표)가 나타날 때까지 노드 위로 마우스 포인터를 이동합니다. 이 단추를 클릭하여 그룹을 확장합니다. 키보드:을 확장 하거나 선택한 그룹 축소를 눌러는 **플러스** 키 (**+**) 또는 **뺀** 키 (**-**).<br />-을 축소 하려면 탭 그룹 노드에 대 한 바로 가기 메뉴를 열고 선택 **그룹**, **축소**합니다.<br />     또는<br />     펼침 단추(위쪽 화살표)가 나타날 때까지 그룹 위로 마우스 포인터를 이동합니다. 이 단추를 클릭하여 그룹을 축소합니다.<br />-모든 그룹을 확장 하려면 **CTRL** + **A** 모든 노드를 선택 합니다. 지도 대 한 바로 가기 메뉴를 열고 **그룹**, **확장**합니다. **참고:** 이 명령이 모든 그룹을 확장할 때 사용할 수 없는 맵 또는 메모리 문제를 생성 하는 경우에 제공 되지 않습니다. 관심 있는 세부 수준으로만 맵을 확장하는 것이 좋습니다.<br />-모든 그룹을 축소 하려면 노드 또는 해당 맵에 대 한 바로 가기 메뉴를 엽니다. 선택 **그룹**, **모두 축소**합니다.|
|네임스페이스, 형식 또는 멤버에 대한 코드 정의를 참조하세요.|노드에 대 한 바로 가기 메뉴를 열고 **정의로 이동**합니다.<br /><br /> 또는<br /><br /> 노드를 두 번 클릭합니다. 확장된 그룹의 경우 그룹의 헤더를 두 번 클릭합니다.<br /><br /> 또는<br /><br /> 노드 및 키를 눌러 선택 **F12**합니다.<br /><br /> 예를 들어:<br /><br /> -하나의 클래스를 포함 하는 네임 스페이스에 대 한 클래스의 코드 파일이 열리고 해당 클래스의 정의 표시 합니다. 다른 경우에는 **기호 찾기 결과** 창 코드 파일의 목록을 표시 합니다. **참고:** Visual Basic 네임 스페이스에서이 작업을 수행 하면 네임 스페이스 뒤 코드 파일이 열리지 않습니다. 선택 된 노드는 Visual Basic의 네임 스페이스를 포함 하는 그룹에이 작업을 수행 하는 경우에이 문제가 발생 합니다. 이 문제를 해결하려면 수동으로 네임스페이스를 뒤의 코드 파일을 찾거나 선택 항목에서 네임스페이스에 대한 노드를 생략합니다.<br />-는 클래스 또는 partial 클래스에 대해 해당 클래스의 코드 파일이 열리고 클래스 정의 표시 합니다.<br />-는 메서드에 대 한 부모 클래스의 코드 파일이 열리고 메서드 정의 표시 합니다.|
|집계 링크에 참여하는 종속성 및 항목을 검사합니다.|관심이 및 선택을 위해 바로 가기 메뉴를 열고 링크를 선택 합니다. 선택 **영향을 주는 링크 표시** 또는 **영향을 주는 링크 새 코드 맵에 표시**합니다.<br /><br /> 그룹이 링크의 양 끝에서 확장되고 링크에 참여하는 항목 및 종속성만 표시됩니다. **참고:** 부분 그룹에 있는 항목 간의 종속성을 검사 하는 경우이 동작이 수행 될 수 있습니다. <ul><li>이러한 링크는 여전히 존재 하는 경우에 프로그램 검사에 참여 하지 않는 항목에 대 한 링크는 지도에서 사라집니다.</li><li>부분 그룹에 있는 항목의 링크를 검사한 후 나중에 동일한 항목의 다른 링크를 검사한다고 가정합니다. 두 번째 검사 중에 대상 부분 그룹은 첫 번째 검사의 항목만 표시합니다. 첫 번째 검사 프로그램에 참여 하지만 두 번째 검사 프로그램에 참여 하지 않은 있는 대상 항목과 링크 표시 되지 않습니다.</li></ul> 선택 그룹에서 누락 된 항목을 보려면 **자식 다시 페치**![자식 다시 페치 아이콘](../modeling/media/dependencygraph_deletednodesicon.png) (나타내는 그룹의 일부 구성원을 지도에 표시 되는지). 사용자의 작업을 실행 취소를 시도할 수도 있습니다 (키보드: 키를 눌러 **CTRL + Z**) 새 맵에 종속성을 확인 하 고 있습니다.|
|서로 다른 그룹의 여러 노드 간에 종속성을 검사합니다.|모든 자식을 볼 수 있도록 그룹을 확장합니다. 자식을 포함하여 관심 있는 노드를 모두 선택합니다. 맵에 선택한 노드 간의 그룹 간 링크가 표시됩니다.<br /><br /> 그룹의 모든 노드를 선택 하려면 **SHIFT** 및 해당 그룹 주위의 사각형을 그리는 동안 마우스 왼쪽된 단추입니다. 지도에 있는 모든 노드를 선택 하려면 **CTRL**+**A**합니다. **팁:** 항상 그룹 간 링크를 표시 하려면 선택 **레이아웃** 맵 도구 모음의 **고급**, **그룹 간 링크 모두 표시**합니다.|
|노드 또는 링크가 참조하는 항목을 확인합니다.|노드에 대 한 바로 가기 메뉴를 열고 **모든 참조 찾기**합니다. **참고:** 이 경우에만 적용 됩니다는 `Reference` 노드나 링크에 대 한 특성이 설정 되어 지도의.dgml 파일입니다. 노드 또는 연결에서 항목에 대 한 참조를 추가 하려면 참조 [DGML 파일을 편집 하 여 사용자 지정 코드 맵을](../modeling/customize-code-maps-by-editing-the-dgml-files.md)합니다.|

##  <a name="HidingShowing"></a> 노드 및 링크를 표시 또는 숨기기
 노드를 숨기면 레이아웃 알고리즘에 참여하지 못합니다. 기본적으로 그룹 간 링크는 숨겨져 있습니다. 그룹 간 링크는 서로 다른 그룹에 있는 노드를 연결하는 개별 링크입니다. 그룹이 축소되어 있을 때 맵에서는 모든 그룹 간 링크가 하나의 그룹 간 링크로 집계됩니다. 그룹을 확장하고 그룹 내의 노드를 선택하면 그룹 간 링크가 나타나고 해당 그룹 내의 종속성이 표시됩니다.

> [!CAUTION]
>  Visual Studio Enterprise로 만든 맵을 Visual Studio Professional 사용자와 공유하기 전에 다른 사용자에게 표시할 노드 또는 그룹 간 링크를 숨기기 취소해야 합니다. 그렇지 않으면 해당 사용자가 해당 항목을 숨기기 취소할 수 없습니다.

### <a name="to-hide-or-show-nodes"></a>노드를 숨기거나 표시하려면

|**대상**|**다음이 단계를 수행 합니다.**|
|------------|-----------------------------|
|선택한 노드를 숨깁니다.|1.  숨길 노드를 선택합니다.<br />2.  선택한 노드 또는 맵에 대한 바로 가기 메뉴를 엽니다. 선택 **선택**, **선택 항목 숨기기**합니다.|
|선택하지 않은 노드를 숨깁니다.|1.  계속해서 표시하려는 노드를 선택합니다.<br />2.  선택한 노드 또는 맵에 대한 바로 가기 메뉴를 엽니다. 선택 **선택**, **선택 되지 않은 숨기기**합니다.|
|숨겨진 노드를 표시합니다.|-그룹 내의 모든 숨겨진된 노드는 표시를 먼저 확인의 그룹을 확장 합니다. 바로 가기 메뉴를 열고 **선택**, **자식 숨기기 취소**합니다.<br />     또는<br />     클릭는 **자식 숨기기 취소**![자식 숨기기 취소 아이콘](../modeling/media/dependencygraph_filtericon_hiddennodes.gif) (이 표시 된 숨겨진된 자식 노드가 없는 경우에) 그룹의 왼쪽된 위 모퉁이에서 아이콘입니다.<br />-모든 숨겨진된 노드를 표시 하려면 지도 또는 노드에 대 한 바로 가기 메뉴를 열고 선택한 **선택**, **모두 숨기기 취소**합니다.|

### <a name="to-hide-or-show-links"></a>링크를 숨기거나 표시하려면

|**대상**|**맵 도구 모음에서 레이아웃을 선택, 고급 및 선택**|
|------------|----------------------------------------------------------------------|
|그룹 간 링크를 항상 표시합니다.|**모든 그룹 간 링크 표시**합니다. 그룹 간 집계된 링크를 숨깁니다.|

|**대상**|**다음이 단계를 수행 합니다.**|
|------------|-----------------------------|

##  <a name="AddRemoveNodesLinks"></a> 추가, 제거 또는 노드, 링크 및 주석 이름 바꾸기
 맵을 드릴다운하거나 단순화하기 위해 맵에 더 많거나 적은 항목을 표시할 수 있습니다. 항목 이름을 바꾸고 항목에 주석을 추가할 수도 있습니다.

> [!CAUTION]
>  Visual Studio Enterprise에서 만든 맵을 Visual Studio Professional 사용자와 공유하기 전에 다른 사용자에게 표시할 코드 요소가 맵에 표시되는지 확인합니다. 그렇지 않으면 해당 사용자가 삭제된 코드 요소를 검색할 수 없습니다.

### <a name="add-a-node-for-a-code-element"></a>코드 요소에 대한 노드 추가

|**대상**|**다음이 단계를 수행 합니다.**|
|------------|-----------------------------|

3.  대상 코드 요소를 선택합니다. 두 코드 요소 사이에 링크가 나타납니다.

 \- 또는 -

5.  **RETURN**키를 누릅니다.

2.  키를 눌러 **F2**, 또는 바로 가기 메뉴를 열고 있으며 선택할 **편집**, **이름 바꾸기**합니다.

3.  맵에 편집 상자가 나타나면 코드 요소 또는 링크의 이름을 바꿉니다.

     \- 또는 -

##### <a name="remove-a-code-element-or-link-from-the-map"></a>맵에서 코드 요소 또는 링크 제거

1.  코드 요소 또는 링크 및 키를 눌러 선택 된 **삭제** 키입니다.

     \- 또는 -

     코드 요소 또는 링크에 대 한 바로 가기 메뉴를 열고 **편집**, **제거**합니다.

2.  요소 또는 링크에는 그룹의 일부 이면는 **자식 다시 페치** 단추 ![자식 다시 페치 아이콘](../modeling/media/dependencygraph_deletednodesicon.png) 그룹 안에 표시 됩니다. 누락된 요소와 링크를 검색하려면 이 단추를 클릭합니다.

-   기본 코드에 영향을 주지 않고 맵에서 코드 요소와 링크를 제거할 수 있습니다. 삭제하면 해당 정의가 DGML(.dgml) 파일에서 제거됩니다.

-   DGML을 편집하거나, 정의되지 않은 코드 요소를 추가하거나, 이전 버전의 Visual Studio를 사용하여 만든 맵은 이 기능을 지원하지 않습니다.

##### <a name="flag-a-code-element-for-follow-up"></a>추가 작업을 위해 코드 요소에 플래그 지정

-   기본적으로 코드 요소에는 빨간색 배경이 지정됩니다. 고려 [주석을 추가](#AddComments) 적절 한 추가 정보를 사용 하 여 합니다.

-   요소의 배경색을 변경 하거나 선택 하 여 추가 플래그를 선택 취소 **편집**, **기타 플래그 색**합니다.

##  <a name="ChangeStyleCodeOrLink"></a> 코드 요소 또는 링크의 스타일 변경

6.  에 **색 집합 선택** 또는 **아이콘 집합 선택** 대화 상자에서 다음 중 하나를 수행 합니다.

    |코드 요소 삭제|**삭제**|
    |스타일을 위로 이동|**위로 이동**|
    |코드 요소를 아래로 이동|**아래로 이동**|

##  <a name="CopyLegend"></a> 한 맵에서 스타일 복사

1.  있는지 확인은 **범례** 소스 맵을에 상자가 나타납니다. 맵 도구 모음에 표시 되지 않으면 클릭 **범례**합니다.

2.  에 대 한 바로 가기 메뉴를 열고는 **범례** 상자입니다. 선택 **범례 복사**합니다.

3.  범례를 대상 맵에 붙여넣습니다.

##  <a name="MergeMaps"></a> 코드 맵 병합
 맵 간에 코드 요소를 복사 및 붙여넣어 맵을 병합할 수 있습니다. 코드 요소 식별자가 일치하는 경우 코드 요소 붙여넣기는 병합 작업처럼 동작합니다. 이 작업을 쉽게 하려면 각 어셈블리 또는 이진 파일의 전체 경로가 병합할 각 맵에서 동일하도록 시각화할 모든 어셈블리 또는 이진 파일을 동일한 폴더에 저장합니다.

 또는 이러한 어셈블리 또는 이진 파일을 해당 폴더에서 동일한 맵으로 끌어올 수 있습니다.
