# <a name="ca0063"></a>CA0063
규칙 집합 파일 또는 해당 종속 규칙 집합 파일 중 하나를 로드 하지 못했습니다.

 규칙 집합 파일 또는 해당 종속 규칙 집합 파일 중 하나를 로드 하지 못했습니다.

 지정 된 규칙 집합을 찾을 수 없습니다 또는 규칙 집합에 포함 된 규칙 집합 파일 중 하나를 찾을 수 없습니다. 모든 ruleset 규칙 집합에 포함 된 디스크에 존재 하는지와 적절 한 규칙 집합 디렉터리를 통해 프로젝트에 지정 되 고 되어 있는지 확인은 **CodeAnalysisRuleSetDirectories** 속성 [!INCLUDE[vstecmsbuild](../extensibility/internals/includes/vstecmsbuild_md.md)]합니다.

 이 오류를 디버깅 하려면 텍스트 편집기에서 규칙 집합 파일을 검사 합니다. 규칙 집합 파일의 경로 확인 하려면 솔루션 탐색기에서 프로젝트를 마우스 오른쪽 단추로 클릭 **속성**, 클릭 하 고 **코드 분석**합니다. 규칙 집합 파일을 선택 해야 **이 규칙 집합 실행**합니다. 규칙 집합의 경로를 설명 필드에 나열 됩니다.

 검사는 **경로** 특성 모든 값의 **Include** 요소입니다. 포함 경로 절대 경로 및 환경 변수, 부모/현재 규칙 집합 파일에 상대 경로 사용할 수 있습니다. 예:

```
<Include Path="%PUBLIC%\Documents\RuleSets\alldesign.ruleset" Action="Default" />
<Include Path="..\alldesign.ruleset" Action="Default" />
<Include Path="C:\Rulesets\alldesign.ruleset" Action="Default" />
```

 이러한 각 검사 경로 포함 하 고 모두 유효한 지 확인 합니다.

 경우에 따라 규칙 집합의 위치에 종속 될 수 있습니다는 [!INCLUDE[vstecmsbuild](../extensibility/internals/includes/vstecmsbuild_md.md)] 속성입니다. [!INCLUDE[vstecmsbuild](../extensibility/internals/includes/vstecmsbuild_md.md)] 속성은 규칙 집합에서 참조될 수 없습니다. 이 문제를 해결 하려면에 추가 검색 경로 지정 된 **CodeAnalysisRuleSetDirectories** 속성 [!INCLUDE[vstecmsbuild](../extensibility/internals/includes/vstecmsbuild_md.md)]합니다. 이 시나리오에서는 ruleset의 이름만 지정는 **경로** 특성에는 **Include** 요소입니다.
